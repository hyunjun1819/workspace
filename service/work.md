1. 롤라(LoRA) 모델 사이트 및 워크플로우 연동
사이트 내 LoRA 설명 및 검색 프로세스
LoRA 가이드 페이지: 사이트 몇 페이지에 걸쳐 LoRA가 무엇인지 상세히 설명하는 섹션을 추가합니다.

검색 및 필터링: 사용자가 원하는 모델을 생성하기 위해 사이트 내에서 LoRA를 검색할 때, 필터 값을 설정하여 본인이 쓰고자 하는 베이스 모델에 맞는 LoRA를 찾을 수 있도록 가이드합니다.

탑재 요청: 사용자가 외부(Civitai 등)에서 마음에 드는 LoRA를 발견하면 해당 링크나 파일을 사이트에 올립니다. 이를 확인하여 운영 측에서 서버(ComfyUI)에 해당 LoRA를 직접 탑재해 주는 프로세스입니다.

ComfyUI 동시 사용 및 저장 로직
개별 작업의 독립성: 여러 사용자가 동시에 접속했을 때, 각자 다른 LoRA를 쓰고 싶다면 브라우저 창을 별도로 열어서 작업하면 됩니다.

로컬 vs 서버 반영: 사용자가 노드 위에서 LoRA를 변경하거나 설정을 바꿔도, 그 상태는 자기 로컬(브라우저)에서만 표시됩니다. 다른 사용자의 작업에 즉각 영향을 주지 않습니다.

저장(Save)의 위험성: 변경 후 '저장'을 누르는 순간, 서버에 저장된 공통 워크플로우 파일 자체가 바뀌어 버립니다. 이 문제를 방지하기 위해 저장 버튼 사용에 주의가 필요합니다.

폴더 기반 관리: 해결책으로 워크플로우 폴더를 계정별/이름별로 생성합니다. (예: '한상준' 폴더). 이렇게 하면 한 사용자가 수정한 제이슨(JSON) 워크플로우가 다른 사람의 파일에 영향을 주지 않고 독립적으로 관리됩니다.

로그인 및 백엔드 연동
간이 로그인 구현: 거창한 DB 없이도 Supabase(수파베이스) 등을 활용해 아이디/비번 테이블 하나 정도로 간단하게 로그인 기능을 추가합니다.

데이터 호출 로직: 사용자가 로그인하면 백엔드에서는 해당 사용자의 폴더를 분석합니다. 그중 가장 최근 날짜/시간의 최종 제이슨 파일을 자동으로 불러와서 작업 환경을 세팅해 줍니다. 이는 프론트엔드와 백엔드를 연동하여 각 사용자의 데이터만 독립적으로 관리하기 위함입니다.

2. AI 프롬프트 위저드(마법사) 및 모델별 상세 설정
지원 모델 라인업 및 가이드
대상 모델: Kling(클링) 3.0, 1.0, 1.5, LTX-Video(LTX-2), Veo 3, Runway 등을 모두 포함합니다.

공식 가이드 준수: 각 영상 모델마다 제조사에서 권장하는 공식 프롬프트 가이드가 있습니다. 이를 시스템 프롬프트에 녹여내어 모델별 최적의 결과물을 뽑아내도록 합니다.

핵심 요소 8가지: 카메라 무빙, 조명, 무드, 피사체의 움직임, 카메라와 피사체의 상관관계, 영상 스타일 등 핵심 요소를 포함하여 프롬프트를 구성합니다.

이미지 기반 생성 (I2V) 및 비전 AI
비전 AI 작동: 이미지 업로드는 선택적 옵션입니다. 이미지가 업로드되면 **비전 AI(Gemini 3 Flash)**가 작동하여 이미지의 정보를 분석합니다.

정교한 분석: 조명 상태, 시간대, 객체의 인종, 연령대, 외모 디테일 등 비전 정보를 파악하여 영상 생성을 위한 텍스트로 변환합니다. 이때 시스템 프롬프트는 "영상 활용을 위한 분석"이라는 목적을 명확히 가져야 합니다.

텍스트 기반 생성 (T2V) 및 위저드 질문
부족한 정보 수집: 사용자가 "30대 남자가 복도를 달려간다"처럼 단순하게 입력하면, AI 마법사가 부족한 정보를 단계별로 묻습니다.

스타일 질문: 시네마틱, 애니메이션 등 어떤 스타일을 원하는지 묻고 예시를 보여줍니다.

외형 디테일: 인종, 연령대, 구체적인 생김새를 편하게 작성하도록 가이드합니다.

환경 설정: 조명과 카메라 워킹에 대해 추가로 질문하여 완성도를 높입니다.

JSON 구조를 활용한 핀포인트 수정
구조화된 데이터: 결과물을 일반 텍스트뿐만 아니라 JSON 구조로 생성합니다. 이는 추후 특정 부분만 수정하기 위함입니다.

수정 프로세스: 영상 결과물이 마음에 들지 않을 경우, 사용자가 해당 JSON 값을 '프롬프트 수정기' 탭에 넣습니다.

자연어로 "빨간 머리를 노란 머리로 바꿔줘"라고 입력하면, AI가 JSON 구조 내에서 헤어 스타일 관련 키값만 찾아 정확하게 수정합니다.

전체 프롬프트를 다시 짤 필요 없이 **특정 포인트(Pinpoint)**만 교체하여 다시 생성할 수 있어 효율적입니다.

AI 채팅 및 시스템 프로포터(Persona)
워크플로우 설명: 사용자가 자신의 ComfyUI 제이슨 파일을 업로드하면, AI가 현재 어떤 모델과 어떤 노드를 쓰고 있는지 기술적으로 분석해 설명해 줍니다.

수치 해석: 특정 노드의 수치값이 어떤 변화를 주는지 물어보면 즉각 답변합니다.

페르소나 설정: AI의 페르소나는 **"기술적인 내용을 비전문가(비개발자)에게 영상 용어를 사용하여 쉽게 설명해 주는 전문가"**여야 합니다. 캡처 이미지나 텍스트 복사 붙여넣기를 통해 질문할 수 있는 기능을 포함합니다.
